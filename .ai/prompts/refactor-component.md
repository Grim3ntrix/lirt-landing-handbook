# Purpose

Prompt template for refactoring components.

# Scope

Use this template when improving component structure.

# Instructions

Reference these documents:
1. `.ai/handbook/05-component-architecture.md`
2. `.ai/handbook/01-principles.md`

# Template

Refactor the component:

## Extract Patterns
- Pull out custom hooks for reusable logic
- Create shared UI components
- Separate presentation from logic

## Simplify
- Remove unnecessary complexity
- Reduce to single responsibility
- Clear naming for all parts

## Consistency
- Follow established patterns
- Match existing component style
- Use shared utilities

## Validate
- All tests still pass
- Handbook standards checked
- No behavior change
